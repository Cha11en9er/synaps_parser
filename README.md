# synaps_parser

Парсер карточек организаций Synaps и выгрузка полей в Google Таблицу.

## Окружение (Windows, PowerShell)

Если виртуальное окружение в папке **`.venv`** (с точкой):

```powershell
cd C:\repos\YouDo\18.04.26-synaps_parser
.\.venv\Scripts\Activate.ps1
```

Если PowerShell ругается на политику выполнения:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

Без активации venv можно вызывать интерпретатор напрямую:

```powershell
.\.venv\Scripts\python.exe sheet_sync.py
```

Зависимости: `pip install -r requirements.txt`. Для Playwright после установки пакетов: `playwright install chromium`.

## Переменные `.env`

Нужны учётные данные Google (путь к JSON сервисного аккаунта), `SHEET_ID`, доступ к Synaps (`MAIN_URL`, `MAIL`/`EMAIL`, `PASS`/`PASSWORD`) — как ожидает код в `sheet_sync.py` и `synaps_browser.py`.

## `sheet_sync.py` — таблица → Synaps → таблица

| Режим | Команда |
|--------|---------|
| **Без окна браузера** (по умолчанию, headless) | `python sheet_sync.py` |
| **С окном браузера** | `python sheet_sync.py --headed` |
| **Сохранить DOM** (3 HTML на каждую ссылку Synaps в столбце A) | `python sheet_sync.py --dump-dom` |
| Комбинация | `python sheet_sync.py --headed --dump-dom` |

Флаги:

- **`--headed`** — показать Chromium (удобно для отладки и капчи).
- **`--dump-dom`** — для **каждой уникальной ссылки** из столбца A (карточки Synaps) сохраняются три файла: главная после всех кликов (`__01_main_after_actions.html`), ОКВЭД (`__02_okved.html`), исполнительное производство (`__03_ispolnitelnoe.html`). Имена: `NN_slug_тег__…`, где `NN` — порядок в обходе, `тег` — короткий хэш **исходной** ссылки из таблицы (чтобы две формы одной карточки не перезаписывали друг друга). Каталог по умолчанию — `dom_dumps/` рядом со скриптом; переопределение: переменная окружения **`SYNAPS_DOM_DUMPS_DIR`** (абсолютный или относительный путь). Перед сохранением ждётся сеть (`networkidle`) и типичные блоки страницы — чтобы в HTML попал контент SPA, а не пустая оболочка.

Раньше при `--dump-dom` обходились только строки, где таблица ещё «пустая» по правилу парсинга — из‑за этого при трёх ссылках в листе мог сохраниться DOM только одной организации. Сейчас при `--dump-dom` в браузер попадают **все** уникальные ссылки Synaps из столбца A; заполнение пустых ячеек по-прежнему только для строк, которым реально нужен парсинг.

Если нужно только снять HTML по всем ссылкам, а строк для обновления нет, скрипт всё равно откроет карточки и сохранит DOM (запись в таблицу не выполняется).

## `synaps_browser.py` — локальный тест (JSON + DOM)

По умолчанию браузер **видимый** (окно открыто). Чтобы запускать **без окна**:

```powershell
python synaps_browser.py --headless
```

Чтобы **не** сохранять HTML в `dom_dumps/`:

```powershell
python synaps_browser.py --no-dom
```

Список URL через запятую — по каждому выполняется полный обход (как в таблице): три HTML в `dom_dumps/`, если не указан `--no-dom`.

```powershell
python synaps_browser.py --urls "https://...,https://...,https://..."
```

Параметры `--rounds` и `--sleep-between-rounds` повторяют полный проход по списку URL указанное число раз (каждый раунд снова открывает карточки и перезаписывает JSON последним результатом).

Кратко:

| Нужно | Флаг |
|--------|------|
| Окно браузера | ничего не указывать (по умолчанию headed) |
| Без окна | `--headless` |
| Без файлов DOM | `--no-dom` |

## Старые имена файлов в `dom_dumps/`

Файл вида `organizacii_1226900006101.html` относится к старой логике (один снимок). Сейчас имена вида `01_slug_abc12ef3__01_main_after_actions.html` (порядок, slug, хэш исходного URL из таблицы).
